결함 유형
기능 결함 : 프로그램 또는 시스템의 동작 또는 기능상 결함
데이터 결함 : 계산 또는 출력 값의 결함
성능 결함 : 하드웨어 자원 또는 응답 시간과 관련된 결함
보안 결함 : 보안 기준을 준수하지 못해 발생한 결함
UI 결함 : UI 검증 처리와 관련한 결함
인터페이스 결함 : 시스템 간 또는 기능 간 연동과 관련된 결함
요구사항 결함 : 요구사항 불명확 또는 모순으로 발생한 결함
기타 유형 : 유형 구분이 어렵거나 그 외 유형의 결함

결함 심각도
치명적 결함 : 어플 작동중지, 시스템 운영 중지, 데이터 손실또는 훼손
주요 결함 : 주요 기능 작동 중지 및 불정확한 작동, 대안기능이 없음, 요구되는 성능에 미치지 못함, 필수적인 예외상황 처리가 부적절하거나 누락됨
사소한 결함 : 미미한 수준이거나 다른 대안이 있음, 기능이 대부분 잘 동작하나 일부 불완전, 기능 수행은 가능하나 사용자에게 혼동 및 불편 유발, 화면 표준 불일치 및 미관상 거슬림 사소한 타이핑 오류 등
개선 사항 : 업무 진행상 전혀 문제가 없으나 개선이 필요한 사항

우선순위
긴급 : 즉시 수정
높음 : 높은 결함
보통 : 수정에 다소 시간적 여유가 있음
낮음 : 다른 조건을 고려할 때 수정이 꼭 필요하지 않은 결함

결함 관리 지표 종류
(결함 유형, 결함 심각도,결함 수정 우선순위, 단위 모듈당 결함 수, 개발 단계별로 발견된 결함 수, 현재 시점에서 수정되지 않은 결함 수, 현재 시점에서 수정이 유보된 결함 수)
